Array.map()

Suppose if you have a list of names with first and surname in an array and you want to just get only the first names in that array then in programming it is very common to iterate through that Array's contents, apply a function say get only first name to each item, and create a new array containing the results for all items. With map() method this is possible and we dont need a loop anymore and dont have to add data to the array anymore. With map method the number of output elements is always equal to the input.

Array.filter()


What if one wnat to transform some of the items in an array? With map() method it is not possible because it transforms all the items. For this purpose what one use is filter() method. However, one important difference is filter callback should be a boolean, whether to include original value in the result (true) or to leave it out (false). As one can see logically map and filter method can be chained.

Array.reduce()


OK you know filter can remove items, but what if one wants to add item to the output? For this reduce() method is used, despite what the name of this method implies. Unlike the map() and filter() methods reduce() always returns the array.

So before closing I can explain how recursion compares to iteration.

Roughly speaking, recursion and iteration repeats the same kind of task many times.

Iteration: keep repeating until a task is done. Using a simple for loop to display the numbers from one to ten is an iterative process.

Recursion is a functions calls itself, but with changed or reduced arguments. The recursive strategy of problem solving often yields elegant code more readable and clear compared to other approaches. However, it has one shortcoming, its memory consumption to each recursive calls. An unbounding increasing of memory will stop the ability to execute as we run out of stack or what is called stack overflow.
