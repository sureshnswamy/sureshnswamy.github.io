What 'scope' is, and how it works in javascript?

Scope refers to the variables declaration location and its accessibility at various parts of the program. JavaScript has two scopes – global and local. Any variable declared outside of a function belongs to the global scope, and is therefore accessible from anywhere in your code. Each function has its own scope, and any variable declared within that function belongs to local scope, is only accessible from that function and any nested functions. Because local scope in JavaScript is created by functions, it’s also called function scope. When we put a function inside another function, then we create nested scope.

Use of parentheses ()

Basically the parentheses are a convention to denote a function call in javascript. Also it is used to surround conditional statements, for grouping to enforce Order of Operations and to send and receive parameters.

Function declaration example:

function example ( ) {
return value;
}

Function expression example:

var a = function example () { //named function expression
return value;
}

var a = function ()) { //anonymous function expression
return value;
}

Use of brackets[ ]

Square brackets are typically mostly used for accessing the properties of an Object (or the elements of an Array), so mylist[2] fetches the third element in the Array.

Use of braces { }

The curly braces are used to define the start and end of the function, they also separate conditional blocks, loops etc, within the function. Also it is used for arrays with objects or declaration of Object Literals.

Use of single and double quotes

Whether it is single or double it is used to declare a string in Javascript. It is a programmer preference but has to be consistent. However, Double quotes eliminate the need to escape apostrophes (as in contractions). Consider the string: "I'm Suresh", vs. the otherwise escaped version: 'I\'m Suresh'.

JASON strings must be double quoted.
